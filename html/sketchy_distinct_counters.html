Background

The Distinct Counting Problem

Given a multi-set of n integers from universe U = {0, 1, …, U−1}, we want to report the count of distinct elements in a single pass using sublinear space. This is the core problem behind analytics on massive data streams — counting unique visitors, unique IP addresses, unique queries, etc.

Why Exact Counting is Hard

An exact solution requires at least ⌈log₂ C(U,σ)⌉ > σ bits by a reduction from the membership problem. This means any single-pass exact algorithm needs Ω(σ) space — so we turn to approximation.

Impossibility: No single-pass algorithm can solve distinct counting in o(σ) space exactly. Proof: if such an algorithm existed, we could solve membership (set membership queries) in sublinear space — a contradiction since there are C(U,σ) possible sets.

Exact Approaches

SortingO(n log n) time, O(n log n) bits
Balanced BSTO(n log σ) time, O(σ log U) bits
Bit ArrayO(n) time, U bits space
Hash TableO(n) expected, O(σ log U) bits

Section 01

Basic Flajolet-Martin Sketch

The core intuition: for σ uniformly random numbers in (0,1], the minimum value is approximately 1/(σ+1). So we can estimate σ ≈ 1/X − 1, where X is the minimum observed hash value.

Basic FM Sketch Idealized
1.Let h : U → (0,1] be a uniform random hash function
2.Compute X = min over all i: h(A[i])
3.Output σ̂ = 1/X − 1 as the estimate

Why Does This Work? (Intuition)

If we have σ distinct elements, each hashed uniformly to (0,1], they divide the interval into σ+1 roughly equal parts. The minimum of σ uniform [0,1] samples has expected value 1/(σ+1). Inverting gives us σ.

E[X] = ∫₀¹ Pr[X > z] dz = ∫₀¹ Pr[h(A[i]) > z, ∀i] dz = ∫₀¹ (1−z)^σ dz = 1/(σ+1) ← correct in expectation! Var(X) = σ / [(σ+1)²(σ+2)] < 1/(σ+1)² ← std dev ≈ mean (too high!)
Problem: The standard deviation is approximately equal to the mean — zero is just one standard deviation away! A single unlucky hash value can wildly skew the estimate.

Section 02

Improved FM — Variance Reduction

To reduce variance, we run r independent copies of Basic FM (each with its own hash function) and average the X values. By Chebyshev's inequality, the probability of being more than ε/(σ+1) away from the mean drops to 1/(rε²).

Improved FM Sketch r hash functions
1.Choose r independent hash functions hⱼ : U → (0,1]
2.For each j: compute Xⱼ = min h_j(A[i])
3.Compute mean: X̄ = (ΣXⱼ)/r
4.Output σ̄ = 1/X̄ − 1
E[X̄] = 1/(σ+1) ← still correct in expectation Var(X̄) < 1/(r·(σ+1)²) ← variance drops by factor r! Pr[|X̄ − 1/(σ+1)| ≥ ε/(σ+1)] < 1/(rε²) ← Chebyshev bound To get failure probability < δ: set r = 1/(δε²)

Section 03

Idealized FM — Median of Means

To get failure probability down to δ while using only O(log(1/δ)) instead of O(1/δ) hash functions, we use the classic median-of-means trick: create s copies of Improved FM and take the median estimate.

Idealized FM Sketch Median of Means
1.Create s = ⌈48 ln(1/δ)⌉ copies of Improved FM, each with r = ⌈3/ε²⌉
2.Let σ̄ⱼ = 1/X̄ⱼ − 1 be the j-th estimate
3.Output σ* = median(σ̄₀, …, σ̄_{s-1})
Why does median help? Each Improved FM instance succeeds (within ±ε·σ) with prob ≥ 2/3 (since rε² ≥ 3) If median is bad → at least s/2 instances are bad By Chernoff: Pr[≥ s/2 failures] ≤ e^{-E[Y]/32} ≤ e^{-s/48} ≤ δ (since s ≥ 48 ln 1/δ) Space: s·r hash functions, each needing O(1) real numbers Total: O(ε⁻² log(1/δ)) real numbers

Section 04

Non-Idealized Flajolet-Martin Algorithm

Real hash functions output integers, not real numbers. The FM algorithm uses a bitmap of length L to discretize. We track the trailing zero patterns of hash codes — this encodes roughly log₂(σ) information.

Key Definitions

For integer u, let ρ(u) = position of the least significant 1-bit (i.e., count of trailing zeros). For example: ρ(8) = ρ(1000₂) = 3, ρ(19) = ρ(10011₂) = 0, ρ(0) = L.

Pr[ρ(h(u)) ≥ k] = Pr[bit k-1 of h(u) is 0 AND ... AND bit 0 is 0] = (1/2)^k ← for uniform hash So MAP[k] stays 1 (untouched) with prob ≈ (1 − 1/2^{k+1})^σ MAP[k] gets set to 0 with prob ≈ 1 − (1 − 1/2^{k+1})^σ For k << log₂ σ: MAP[k] ≈ 0 (almost certainly) For k >> log₂ σ: MAP[k] ≈ 1 (almost certainly) Transition at k ≈ log₂(φ·σ), where φ = 0.77351
Non-Idealized FM Algorithm Practical
1.Initialize bitmap MAP[0..L-1] = all 1s
2.For each element u: compute index = ρ(h(u)), set MAP[index] = 0
3.Let R = smallest index such that MAP[R] = 1
4.Output σ̂ = 2^R / φ, where φ = 0.77351
Edge Cases: If MAP is all 0s (R = L), the estimate overflows — σ may be larger than 2^L/φ. If MAP is all 1s (R = 0), estimate is 1/φ ≈ 1.3, even for empty streams.

Section 05

LogLog Algorithm

Instead of one big bitmap, LogLog uses m = 2^k buckets. The first k bits of each hash code select a bucket; the remaining bits record the longest run of leading zeros. The estimate averages (geometric mean) the per-bucket maximums.

Key Insight: Bucketing = Multiple Independent Estimators

By splitting the hash into bucket index (first k bits) and trailing-zero count (remaining bits), we effectively simulate m independent FM sketches — but with just a single hash function!

For each element u with b-bit hash h(u) = y₀y₁...y_{b-1}: bucket x = decimal(y₀...y_{k-1}) ← first k bits r = min index ≥ k such that y_r = 1 ← first 1-bit in remaining B[x] = max(B[x], r − k) ← longest zero prefix seen r̄ = (1/m) · Σ B[j] ← arithmetic mean of B values σ̂ = α_m · m · 2^{r̄} ← geometric mean of 2^B[j] Standard Error ≈ 1.30/√m To achieve ε error: choose m ≥ 1.69/ε²
LogLog Algorithm Durand-Flajolet 2003
1.Initialize array B[0..m-1] = all 0s, where m = 2^k
2.For each element u: compute b-bit hash h(u)
3.Bucket index x = first k bits of h(u)
4.Find r = index of first 1-bit in remaining (b−k) bits
5.Update: B[x] = max(B[x], r − k)
6.Compute mean: r̄ = (1/m) Σ B[j]
7.Output σ̂ = α_m · m · 2^{r̄}

Section 06

SuperLogLog — Taming Outliers

Large bucket values (outlier rᵢ's) can skew the arithmetic mean dramatically. SuperLogLog adds two rules to mitigate this:

Restriction Rule

Only consider buckets where B[j] ≤ ⌈3 + log₂(U/m)⌉. This removes degenerate cases (e.g., hash collisions with all-zero suffixes) and reduces per-bucket storage from ⌈log₂(b-k)⌉ to ⌈log₂(3 + log₂(U/m))⌉ bits.

Truncation Rule

Drop the largest 30% of B[j] values before computing the mean. This eliminates high-bias outliers. Applied after the restriction rule, the number of remaining buckets must be adjusted.

SuperLogLog Standard Error ≈ 1.05/√m ← 24% better than LogLog! LogLog Standard Error ≈ 1.30/√m Space example: U = 2^30, m = 2^10, b = 64-bit hash Without restriction: each bucket stores up to 54 → 6 bits With restriction: each bucket stores up to 23 → 5 bits (saving 1 bit/bucket = 1 KB total)
